# BioDraw 导出方案设计 v1

## 1. 文档目的

本文档用于明确 BioDraw 第一阶段导出能力的设计边界、输入输出模型和后续实现方向，为导出接口抽象、MP4 目标格式和浏览器播放链路提供统一设计依据。

---

## 2. 设计目标

第一阶段导出设计的目标不是一次完成所有导出能力，而是：

- 明确主导出目标是 MP4
- 明确第一阶段先完成导出架构预留
- 明确导出输入边界
- 为后续真正实现 MP4 导出打下基础

---

## 3. 导出目标排序

### 第一优先级
- MP4（用于 PPT、本地播放器、浏览器播放）

### 第二优先级
- 工程文件 JSON（保存 / 继续编辑）

### 后续优先级
- 网页演示包
- GIF
- 模板包

---

## 4. 第一阶段结论

第一阶段建议：

1. **必须完成工程文件保存**
2. **必须完成导出服务接口抽象**
3. **可以暂不完成真正 MP4 编码实现**
4. **导出按钮与导出上下文模型必须明确**

---

## 5. MP4 目标格式

建议目标格式为：

- 容器：MP4
- 视频编码：H.264
- 音频编码：AAC（如后续需要音频）
- 画面尺寸：基于画布尺寸或导出尺寸配置

---

## 6. 导出输入模型

导出系统建议接收标准化导出上下文，而不是直接耦合 UI 组件。

```ts
type ExportContext = {
  project: ProjectDocument
  width: number
  height: number
  durationMs: number
  backgroundColor?: string
}
```

### 说明
- `project`：完整工程数据
- `width / height`：导出画面尺寸
- `durationMs`：导出总时长
- `backgroundColor`：背景色

---

## 7. 导出服务接口建议

```ts
type ExportService = {
  exportProject(context: ExportContext): Promise<ExportResult>
}
```

### ExportResult 建议

```ts
type ExportResult = {
  success: boolean
  message?: string
  outputPath?: string
}
```

---

## 8. 第一阶段 UI 行为建议

点击“导出”按钮后：

- 调用统一导出入口
- 若 MP4 尚未实现，可先输出提示信息
- 不得将导出逻辑散写在多个组件里

---

## 9. 与播放器的关系

未来导出系统应建立在播放器 / 预览系统之上，因为导出本质上需要：

- 按时间推进
- 逐帧解析场景状态
- 渲染每一帧

因此第一阶段就应确保：
- 场景帧解析逻辑可独立调用
- 导出系统未来可复用播放器时间解析机制

---

## 10. 画面尺寸策略

第一阶段建议：

### 默认策略
- 导出尺寸直接使用当前工程画布尺寸

### 后续可扩展
- 导出前允许选择 720p / 1080p
- 按比例缩放场景内容

---

## 11. 浏览器播放策略

第一阶段浏览器播放不需要单独发明复杂格式，建议：

- 先用编辑器内预览
- 后续导出 MP4 后，浏览器直接播放 MP4

不建议第一阶段投入：
- HTML 动画包
- 内嵌 PPT 特殊对象
- 复杂网页播放器导出

---

## 12. 第一阶段验收标准

导出方案设计与导出架构预留视为完成，至少要满足：

1. UI 中有统一导出入口
2. 导出逻辑通过统一服务接口调用
3. `ExportContext` 输入边界明确
4. 不把导出逻辑散落写进多个 React 组件
5. 后续 MP4 能有明确接入点

---

## 14. 结语

这份导出方案设计文档的目标，是为当前阶段建立一套**清晰的导出边界与实现入口**，而不是提前完成所有导出能力。

后续所有导出实现，都应尽量反复对照下面这句话：

**当前导出设计，是否真的在服务 BioDraw 第一阶段“成果可保存、导出链路可预留”的目标？**

如果没有，就应及时收敛。
